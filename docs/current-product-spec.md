# Hello Lang Current Product Spec

Last updated: 2026-04-24

## 1. Product summary

Hello Lang is a mobile-first Japanese-to-English phrase translation and memorization app.

The product combines three user jobs into one flow:

- turn a Japanese phrase into natural spoken English
- save the phrase instantly as a personal study item
- rehearse saved phrases through flashcards and audio playback

The current application is built with Next.js App Router and is designed primarily for smartphone use.

## 2. Current user flow

### Translation flow

1. User opens the home screen.
2. User enters Japanese text manually or uses browser speech recognition.
3. User taps the translate button.
4. The app sends the text to `/api/translate`.
5. Gemini returns:
   - a natural English phrase
   - a short usage category such as daily life, work, food, or trouble
6. The app immediately:
   - displays the result
   - saves it as a phrase record
   - reads the English phrase aloud with browser speech synthesis

### Study flow

1. User opens the flashcard screen.
2. Non-mastered phrases are loaded into a card deck.
3. User can:
   - swipe right to mark a phrase as mastered
   - swipe left to send a phrase back for review
   - replay audio
   - loop one card
   - autoplay the deck

### Review and management flow

1. User opens the phrase list screen.
2. User can:
   - search by Japanese text
   - search by English text
   - search by category
   - play single audio
   - autoplay the filtered list
   - delete phrases

### Account flow

1. User can sign up or sign in with email and password via Supabase Auth.
2. Logged-out users use local storage.
3. Logged-in users use Supabase-backed cloud storage.
4. Local phrases are migrated to Supabase after login.

## 3. Screens and routes

- `/`
  Primary translation and capture screen.
- `/cards`
  Flashcard study screen for active learning phrases.
- `/list`
  Phrase notebook and search screen.
- `/login`
  Email/password authentication screen.

Bottom navigation is always visible and links to all four routes.

## 4. Core functional specification

### Input

- Primary source language: Japanese
- Primary output language: English
- Input methods:
  - text area
  - browser speech recognition
- Speech recognition settings:
  - locale: `ja-JP`
  - one-shot recognition
  - no interim results

### Translation

- Translation is generated by Gemini through `@google/genai`
- Model currently used: `gemini-2.5-flash`
- Prompt requests:
  - natural and everyday English
  - preservation of raw tone for slang or aggressive language
  - assignment of a short situation category
- Response format is constrained to JSON:
  - `translation`
  - `category`

### Phrase storage

Each phrase record currently contains:

- `id`
- `original`
- `translation`
- `category`
- `status`
- `created_at`
- `user_id` when authenticated

Current statuses:

- `learning`
- `mastered`

### Audio output

- Playback uses browser `speechSynthesis`
- Output language is `en-US`
- Translation screen uses rate `0.9`
- List and card screens use default rate

### Study interactions

- Flashcards exclude `mastered` phrases on first deck build
- Reset restores all phrases to `learning`
- Review swipe rotates a card to the bottom of the deck
- Mastered swipe removes the card from the active deck and updates stored status

### Search and review

- Search is client-side only
- Search fields:
  - original Japanese text
  - lowercased English translation
  - lowercased category

## 5. Data and backend architecture

### Frontend

- Next.js 16 App Router
- React 19
- CSS Modules plus global CSS variables
- Framer Motion for card transitions
- Lucide React for icons

### Backend and integrations

- Next.js route handler at `/api/translate`
- Google Gemini for translation
- Supabase Auth for account management
- Supabase Postgres for phrase persistence

### Database schema

Current `phrases` table:

- `id UUID PRIMARY KEY`
- `user_id UUID NOT NULL`
- `original TEXT NOT NULL`
- `translation TEXT NOT NULL`
- `category TEXT`
- `status TEXT DEFAULT 'learning'`
- `created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP`

### Access control

Supabase Row Level Security is enabled so users can only view, insert, update, and delete their own phrases.

## 6. UX and design specification

The visual system is intentionally warm and non-technical.

- visual theme: terracotta, cream, sage, espresso
- typography: Plus Jakarta Sans and Noto Sans JP
- layout: mobile-first with max width 480px
- fixed bottom navigation for one-hand mobile use
- rounded cards and soft borders instead of heavy shadows
- accessibility intent:
  - WCAG AA contrast targets documented
  - focus color defined
  - minimum readable text sizes documented

## 7. Current strengths

- very fast capture-to-study loop
- no onboarding required for first use
- works without login through local storage
- supports both text input and voice input
- immediate audio reinforcement after translation
- simple card metaphor for memorization

## 8. Current limitations and product gaps

### User and market limitations

- only supports Japanese to English
- no explicit user persona onboarding
- no collaborative or teacher features
- no subscription, paywall, or billing logic

### Product limitations

- no example sentence generation beyond the single translated phrase
- no spaced repetition algorithm beyond mastered/learning status
- no learning streaks, reminders, goals, or gamification
- no phrase editing
- no favorites, tags, folders, or custom lists
- no pronunciation scoring or speaking practice feedback
- no onboarding tutorial
- no analytics or experimentation framework surfaced in the app

### Technical risks

- browser speech recognition support differs by browser and device
- text-to-speech quality depends on system voices
- translation endpoint has minimal validation and no rate limiting
- no visible error recovery flow beyond alerts
- no test suite is present in the current repository
- no production telemetry or monitoring is implemented in-app

## 9. Existing business signals already implied by the repo

The repository already suggests an early SaaS mindset:

- production domain exists: `hellolang.app`
- Vercel deployment metadata exists
- Supabase cloud persistence is already integrated
- a Notion daily report script exists for founder operations
- report config includes user, phrase, and MRR goals

This means the product is beyond prototype aesthetics and is already positioned as an early-stage consumer learning product that can be instrumented and operated like a startup.
